# C2W vs W2W (Chip-to-Wafer vs Wafer-to-Wafer)

> [!def] 定义 (Definition)
> C2W (Chip-to-Wafer) 和 W2W (Wafer-to-Wafer) 是混合键合的两种主要工艺方式，分别指芯片对晶圆和晶圆对晶圆的键合模式。

## 📖 基本原理 (Fundamentals)

### C2W (Chip-to-Wafer)

```
键合流程：
1. 将已知良品芯片(KGD)从晶圆上切割
2. 逐个放置到目标晶圆上进行键合
```

**特点**：
- 可选择良品芯片键合，提高整体良率
- 支持不同尺寸芯片的异构集成
- 工艺复杂度较高，逐个键合效率较低

### W2W (Wafer-to-Wafer)

```
键合流程：
1. 两片完整晶圆对准
2. 整片晶圆一次性键合
```

**特点**：
- 键合效率高，适合大规模生产
- 要求两片晶圆尺寸一致
- 一颗芯片不良会影响整片晶圆

## 📊 对比分析

| 特性 | C2W | W2W |
|------|-----|-----|
| **良率策略** | 先筛选后键合 | 整片键合后筛选 |
| **芯片尺寸** | 可不同 | 必须一致 |
| **生产效率** | 较低 | 高 |
| **适用场景** | 异构集成、高价值芯片 | 同构堆叠、量产产品 |
| **对准精度** | 挑战大（逐个对准） | 相对容易 |
| **成本** | 较高 | 较低 |

## 🔗 与HBI的关系 (Relevance to HBI)

### 在混合键合中的作用

- 两种方式均可实现HBI互连
- 选择取决于应用需求和成本考量
- HBM等高密度存储多采用C2W（需要堆叠多层不同功能的芯片）

### 选择考量

```
选择 C2W 当：
├── 需要异构集成（不同尺寸/功能的芯片）
├── 芯片良率较低，需要筛选
└── 产品价值高，可接受较高成本

选择 W2W 当：
├── 芯片尺寸一致
├── 良率高
└── 追求量产效率和成本优势
```
